RMAN Backup, Restore, and Recovery

Recovery Manager is Oracle's backup and recovery tool. It understands database files, tracks backup metadata, validates recoverability, and automates restore and media recovery work that is risky to do manually.

RMAN

Important Concepts

Backup set and image copy

A backup set is RMAN's compact backup format, usually written as backup pieces. An image copy is a file-level copy that can sometimes be switched into place faster.

Full, incremental, and archived redo backups

A full backup captures datafiles. Incremental backups capture changed blocks. Archived redo logs are required to recover committed changes after the backup time.

Restore vs recover

Restore means copy backup files back to disk. Recover means apply archived redo and incremental backups so the restored files reach the required point in time.

Control file and SPFILE

The control file stores database structure and RMAN metadata. The SPFILE stores initialization parameters. Back them up because disaster recovery often starts with these files.

Recovery catalog

A catalog is an optional separate schema that stores RMAN metadata outside the target database control file. It is useful for longer retention and many databases.

Retention and obsolete backups

The retention policy defines how long backups must remain recoverable. RMAN can report and delete obsolete backups based on that policy.

Connection and configuration

Use these first to connect and inspect backup policy.

rman target /
rman target sys/password@ORCL

show all;

configure retention policy to recovery window of 7 days;
configure controlfile autobackup on;
configure default device type to disk;
configure channel device type disk format '/backup/rman/%d_%U.bkp';
Backup commands

Core commands for database, tablespace, datafile, archive log, and control file backups.

backup database;
backup database plus archivelog;
backup incremental level 0 database;
backup incremental level 1 database;

backup tablespace users;
backup datafile 4;
backup current controlfile;
backup spfile;

backup archivelog all;
backup archivelog all delete input;
Restore commands

Restore copies files from backups. Recovery is usually needed after restore.

startup mount;
restore database;
recover database;
alter database open;

restore tablespace users;
recover tablespace users;

restore datafile 4;
recover datafile 4;

restore controlfile from autobackup;
restore spfile from autobackup;
Point-in-time recovery

Use point-in-time recovery when you need to recover before a mistake such as accidental delete or drop.

run {
  set until time "to_date('2026-05-22 10:30:00', 'yyyy-mm-dd hh24:mi:ss')";
  restore database;
  recover database;
}
alter database open resetlogs;

run {
  set until sequence 120 thread 1;
  restore database;
  recover database;
}
alter database open resetlogs;
Validation and reporting

Validate before a real incident. These commands help prove backups exist and can be read.

list backup summary;
list backup of database;
list backup of archivelog all;

report schema;
report obsolete;
report need backup;

crosscheck backup;
crosscheck archivelog all;
validate database;
restore database validate;
Maintenance and cleanup

Keep RMAN metadata accurate and remove backups only when they are no longer needed by the retention policy.

delete obsolete;
delete expired backup;
delete expired archivelog all;

catalog start with '/backup/rman/';
change backup unavailable;
change backup available;

resync catalog;

Quick Reference

Starter snippets to keep useful Oracle commands close while you build your own examples.

Task Command or Query Why It Matters
Current schema select sys_context('USERENV', 'CURRENT_SCHEMA') from dual; Confirms which schema name resolution is using.
List tables select table_name from user_tables order by table_name; Shows tables owned by the connected user.
Execution plan select * from table(dbms_xplan.display_cursor(null, null, 'ALLSTATS LAST')); Reviews actual plan details after executing a statement.
Tablespace usage select tablespace_name, bytes / 1024 / 1024 mb from user_segments; Helps estimate space used by objects in your schema.
Session info select sid, serial#, username, status from v$session where username is not null; Useful for admin troubleshooting when privileges allow access.

Suggested Study Flow

A compact path for turning scattered learning into a usable Oracle knowledgebase.

1. Capture

After every lab or issue, add one note with the exact command, error, concept, and outcome. Keep notes atomic so they are easy to search later.

2. Connect

Group related notes under architecture, SQL, PL/SQL, admin, and performance. Add cross-references in the note body when one concept explains another.

3. Practice

Convert useful notes into repeatable checks: sample queries, setup scripts, troubleshooting steps, and before/after performance observations.
